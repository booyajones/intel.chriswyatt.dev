# PRD Brief Template

**Purpose:** Get stakeholder confirmation BEFORE drafting. This catches "wrong problem" framing early.

**Keep it dead simple** — this is what stakeholders see first. Half-page maximum.

---

## PRD Brief: [Feature/Problem Name]

**Problem in one sentence:**  
[What is broken or missing, and what is the consequence?]

**Who owns this today:**  
[Which team handles this operationally right now?]

**Who is affected:**  
[Which users or internal teams experience the pain?]

**Scope:**  
[What systems/partners/payment types are in play? Is this one partner or all?]

**Solution direction:**  
[What is the rough approach? Field-level controls? New workflow? Detection logic? API change?]

**Key assumption to validate:**  
[The single biggest assumption that, if wrong, changes everything]  
*e.g., "We assume this requires new detection logic rather than restricting who can edit supplier bank details."*

---

## Stakeholder Confirmation

**Please confirm:**
1. Is the problem statement accurate? If not, what IS the actual problem?
2. Are we solving this for the right person/team?
3. Is anything missing from scope?
4. Who else should be involved in reviewing the full PRD?

**Once confirmed, I'll proceed with drafting the full PRD.**
